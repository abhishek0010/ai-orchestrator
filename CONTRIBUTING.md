# contributing to ai-orchestrator

Welcome! We appreciate your interest in contributing to the `ai-orchestrator` project. This document outlines the guidelines and workflow for contributing to the project.

## Project Philosophy
- **Pure Bash**: Core orchestration is handled via Bash, `jq`, and `curl`. No Python dependencies allowed in the core logic.
- **Portability**: The setup should remain portable and easy to install on any Unix-like system (macOS/Linux).
- **Local-First**: All heavy lifting (coding, reviews, commit messages) is delegated to local Ollama models.

## Development Workflow

### 1. Automated implementation (`/implement`)
For any non-trivial task, use the internal `/implement` command:
- It triggers a pipeline: **Planner** → **Coder** → **Reviewer**.
- It ensures consistency and follows the project's language standards.

### 2. AI-Driven Commits (`local-commit.sh`)
Avoid manual commit messages. Use the local AI to analyze your diffs:
```bash
./scripts/local-commit.sh
```

### 3. Pull Request Process (`open-pr.sh`)
Before opening a PR on GitHub, generate a structured description using:
```bash
./scripts/open-pr.sh
```

## Repository Structure

- **`agents/`**: System prompts/rules for specific agents (Planner, Coder, Reviewer, etc.).
- **`commands/`**: Definitions for slash commands used in IDE agents.
- **`skills/`**: Language-specific coding standards and best practices.
- **`scripts/`**: Core logic and utility scripts.
- **`documentation/`**: Architectural overviews and rules.

## How to Contribute

1.  **Fork** the repository: [Mybono/ai-orchestrator](https://github.com/Mybono/ai-orchestrator)
2.  **Clone** your fork and run `./scripts/install.sh`.
3.  **Create a feature branch** for your changes.
4.  **Use the tools**: Leverage `/implement` for the task and `local-commit.sh` for Commits.
5.  **Submit a Pull Request**: Use the description generated by `open-pr.sh`.

## Code Style
- Keep Bash scripts modular and well-commented.
- Use `jq` for any JSON processing to maintain the project's zero-dependency nature.
- Follow the standards defined in the `skills/` directory for the specific language you are working on.

Thank you for helping make `ai-orchestrator` better!
