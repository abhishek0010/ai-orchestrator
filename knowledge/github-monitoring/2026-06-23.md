## Projects Analyzed

### [open-multi-agent/open-multi-agent](https://github.com/open-multi-agent/open-multi-agent) ⭐ 6422 · TypeScript · Updated: 0 days ago

**What it does:** Provides a goal‑driven multi‑agent framework with a lightweight “consensus” primitive that lets a proposer LLM generate a suggestion and a judge LLM verify it, enforcing a token‑budget invariant.

**Problem it solves for ai-orchestrator:** `src/core/Orchestrator.ts` currently has a simple “run → review → repeat” loop but lacks a reusable, budget‑aware verify step (e.g. when a coder’s output must be judged before acceptance). This forces duplicated code in many agents and makes token‑budget enforcement fragile.

**How to integrate:**

- **File:** `src/core/Orchestrator.ts`
- **Change:**
  1. Add a new `runConsensus<T>(options: ConsensusOptions<T>)` method that (a) calls a proposer agent via `AgentRunner`, (b) calls a judge agent on the proposer’s output, (c) checks the combined token usage against `budget.maxTokens` (existing budget config), and (d) returns the judge‑approved result or throws a `ConsensusError`.
  2. Define a lightweight `ConsensusOptions` interface (proposerRole, judgeRole, input, budget) at the bottom of the file.
  3. Hook this method into `AgentLoop.ts` for the “coder → reviewer” phase, replacing the ad‑hoc reviewer call with `orchestrator.runConsensus({ proposerRole: 'coder', judgeRole: 'reviewer', input: taskContextPath, budget: this.budget })`.

**Effort:** Quick (hours) – adding ~30 lines of TypeScript and a few unit‑test updates.

**Risk:** Low – the method is pure TypeScript, reuses existing agent‑running infrastructure, and only adds optional verification logic.

**Verdict: ADOPT** ✅ – The consensus primitive cleanly abstracts verification, enforces token budgets, and eliminates duplicated reviewer code, fitting naturally into the existing orchestration flow.
