Generate a visual Entity Relationship Diagram (ERD) from an existing database schema or design.

Load the following expertise before starting:

- [Architect Agent](../../../agents/architect.md)

## Process

1. Explore the existing database schema (DDL files, models, or active connection).
2. Map all tables, columns, and relationships.
3. Generate a Mermaid diagram representing the ERD.
4. Add descriptions for each entity and its role in the system.

## Output

A high-quality Mermaid ERD and a summary of the data relationships.

## Rules

- Group related tables into logical namespaces or subgraphs.
- Clearly mark primary keys (PK) and foreign keys (FK).
- Annotate relationship types (e.g., "1 to many").
