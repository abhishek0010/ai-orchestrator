Generate a comprehensive README.md file from project analysis.

Load the following expertise before starting:

- [doc-writer](../../../agents/doc-writer.md)
- [doc-standarts](../../../skills/doc-standarts.md)

## Process

1. Perform a deep analysis of the project:
   - Identify tech stack (`package.json`, `pyproject.toml`, etc.).
   - Find main entry points and directory structure.
   - Detect installation and running procedures.
   - Identify core features and architecture.
2. Structure the README:
   - Clear project title and one-sentence description.
   - Tech stack icons/badges.
   - Quick Start (Installation, Run, Tests).
   - Core Features list.
   - Project Structure overview.
   - Contribution guidelines.
   - License.
3. Apply professional formatting and standard headings.

## Rules

- Strictly follow [doc-standarts](../../../skills/doc-standarts.md).
- Use [humanizer](../../../skills/humanizer.md) for a professional, non-robot voice.
- Ensure all links in the README are functional.
- Include a "Powered by ai-orchestrator" section if appropriate.
