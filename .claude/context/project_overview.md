## Architecture & Conventions
- All agents are markdown files in `agents/` with a YAML front matter block (name, description, tools)
- All slash-commands are markdown files in `commands/` — no front matter, open with an imperative sentence, one `## Instructions` section with numbered steps
- Context files go to `.claude/context/` with naming `task_context_<domain>.md`, `review_<filename>.md`, etc.
- Knowledge files go to `knowledge/` — `context-index.md` is the cross-project index, `outcomes.jsonl` is the append-only learning log, `embeddings.jsonl` is the embedding store, `learning-report-YYYY-MM-DD.md` files are written by learn.sh
- Scripts use bash + jq + python3 (stdlib only) — no external Python packages or Node dependencies in the orchestrator core
- JSON processing always uses `jq`, never manual string manipulation
- Planner writes `project_overview.md` and reads `knowledge/context-index.md` at session start
- doc-writer signals PreToolUse hook via `touch /tmp/.doc_writer_active` before any `.md` write, cleans up after
- File-path handoff between agents: agents pass file paths in prompts, never full content
- PostToolUse hook in `.claude/settings.json` fires on Write when basename contains "review"; calls capture-outcome.sh
- `outcomes.jsonl` first line is a `#` comment describing the schema; subsequent lines are valid JSON objects
- `capture-outcome.sh` auto-triggers `learn.sh --apply` via `& disown` when non-comment line count is a non-zero multiple of 10; also auto-triggers `embed-outcomes.sh` after every capture if Ollama is reachable (curl --max-time 2 guard)
- `embed-outcomes.sh` skips already-embedded records by comparing line counts; --force truncates and regenerates all; uses `grep -c '^{'` to count data lines
- `semantic-search.sh` embeds the query via `/api/embeddings`, loads `embeddings.jsonl`, computes cosine similarity in pure python3 (math + json stdlib only, no numpy), outputs top-K outcomes above min-similarity threshold
- `learn.sh` supports `--query TEXT`; when set, calls `semantic-search.sh` to retrieve similar records and analyses their issues instead of the jq grouping path; falls back to all distinct issues if none meet MIN_COUNT
- Ollama embeddings endpoint: POST `http://localhost:11434/api/embeddings`, body `{"model":"...","prompt":"..."}`, response field `embedding` (array of floats)
- `embeddings.jsonl` schema: `{"index": <int>, "text": "<string>", "embedding": [<float>, ...]}` — one object per line, no comments, compact (jq -cn)
- TypeScript source lives in `src/`; compiled to `dist/`; entry points use NodeNext module resolution (`.js` extensions on all local imports)
- TypeScript strict mode + `noUncheckedIndexedAccess` + `exactOptionalPropertyTypes` — no `any`, explicit return types, `readonly` everywhere
- Error handling in TypeScript returns `{ ok: boolean; ... }` result objects rather than throwing; matches `HttpCallResult` pattern from `LlmHttpClient.ts`
- Integration clients (Telegram, Slack) use zero npm dependencies: `node:http`, `node:crypto`, `node:child_process`, global `fetch`
- Telegram long-polling uses AbortController stored on the class instance so `stop()` can abort an in-flight fetch
- Slack signature verification uses `timingSafeEqual` from `node:crypto`; requires same-length buffer check before calling
- HeadroomBridge availability is cached in module scope (`let _available: boolean | undefined`) — checked once per process lifetime via `execSync`; all compression is async with timeouts via `exec`
- `settings.json` (project root) PostToolUse array: first element is SkillSpector (matcher Write|Edit), second is headroom init (matcher Bash|Write|Edit)

## Do Not Touch
- `README.md`: managed by doc-writer agent only (enforced by PreToolUse hook)
- `llm-config.json` models: can change at runtime — no restart needed; do not hardcode model names in scripts
- `knowledge/outcomes.jsonl`: append-only — never edit by hand, only via `scripts/capture-outcome.sh`
- `knowledge/embeddings.jsonl`: generated output — never edit by hand; re-run embed-outcomes.sh to regenerate
- `knowledge/learning-report-*.md`: generated output — never edit by hand; re-run learn.sh to regenerate
- `src/types/index.ts`: central type registry shared by all modules — **never replace the whole file**; only add new types or fields to existing ones; removing or renaming existing exports breaks every other file
- `src/core/Orchestrator.ts`: complex stateful orchestration — **never replace**; only add methods or extend existing ones
- `src/core/AgentLoop.ts`: autonomous daemon — **never replace**; add/modify individual methods only
- `src/core/ToolRunner.ts`: tool execution layer — **never replace**; add new `case` branches to `execute()` or new private methods

## Known Constraints
- **Never replace an existing TypeScript file wholesale.** Always read the current file first, then write the complete updated content including all pre-existing code. The coder's %%FILE block must contain the full file — not a rewrite from scratch. Replacing files breaks imports across the entire project.
- Never mock Ollama in tests — all agent tests require a live local model
- Scripts must use bash + jq only for JSON; python3 stdlib (math, json, sys) is allowed for numeric computation (cosine similarity) — no external Python packages
- commands/*.md files have no YAML front matter — match create-bug.md style exactly
- knowledge/context-index.md sections are separated by `---` horizontal rules; heading style is title case
- capture-outcome.sh derives REPO_DIR from BASH_SOURCE[0]; never hardcode absolute paths
- The PostToolUse hook uses check=False on subprocess.run so script failures never block Claude's Write operation
- learn.sh must exit 0 silently when outcomes.jsonl is missing or empty — never error in that case
- learn.sh auto-trigger uses disown — never block capture-outcome.sh with wait
- embed-outcomes.sh must exit 0 silently when Ollama is unavailable (curl --max-time 2 check) — never error
- semantic-search.sh exits 1 with a clear message when Ollama is unavailable or embeddings.jsonl is missing
- call_ollama.sh called from learn.sh with `--role reviewer`; LLM failure handled with `|| true`; empty response skips the issue silently
- src/integrations/ uses zero npm dependencies — node:built-ins + fetch only
- Telegram: text chunks at 4096 chars; Slack: text chunks at 3000 chars (API practical limit)
- Slack HTTP server must respond 200 within 3 seconds of receiving an event_callback — respond first, process async
- HeadroomBridge must never block the main path — `compressWithHeadroom` and `getHeadroomStats` must resolve on any error; `isHeadroomAvailable` is synchronous but cached after first call
- `exactOptionalPropertyTypes` is on — optional fields in result types must be **absent** from object literals when not set, not set to `undefined`
